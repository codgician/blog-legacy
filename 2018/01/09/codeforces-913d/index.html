题面

某场考试时长为 T (1≤T≤109)T \ (1 \le T \le 10^9)T (1≤T≤109) 毫秒，并且由 n (1≤n≤2⋅105)n \ (1 \le n \le 2 \cdot 10^5)n (1≤n≤2⋅105) 道题目组成（每题 111 分）。对于第 iii 道题目，你可以选择花费 ti (1≤ti≤104)t_i \ (1 \le t_i \le 10^4)ti​ (1≤ti​≤104) 毫秒来解出它或是直接跳过它。

老师见你很皮，于是认为部分题目对你而言太简单了。所以，对于每一道题目 iii 老师都规定了一个指数 ai (1≤ai≤n)a_i \ (1 \le a_i \le n)ai​ (1≤ai​≤n)，并表示只有当你解出的总题数不超过 aia_iai​ 时第 iii 题才能为你带来得分，否则就算你做出了第 iii 题该题分数也不会计入总分。

你显然希望获得最大分数，那就看你选择做哪几道题咯（如果有多种答案随便输出一种就好）~

题目链接

贪心！

首先，我们假设我们能获得的最大分数为 kkk。不难发现，花费时间去做那些并不能给自己带来分数的题目是毫无意义的。因此，一定存在做出题目数量恰好为 kkk 并且得分为 kkk 的方案。

那么我们就找这种方案就好咯~

首先，我们需要把 kkk 的大小确定下来。

对于任意一个总分 mmm，只要我们能找到 mmm 道题使得它们所耗时间之和小于等于考试总长并且每道题的 aia_iai​ 值都大于等于 mmm，那么这个 mmm 就是可行的。在具体实现过程中仅需要把题目都按花费时间从小到大的顺序排个序，然后依次遍历，只要满足 ai≥ma_i \ge mai​≥m 就做它，如此一直到时间耗尽，最后比对一下所做的题数是否大于等于 mmm 即可。而 kkk 就是最大的可行的 mmm。

在寻找到 kkk 后输出方案就很容易咯（在上述验证 kkk 可行性的过程中实际上都已经把方案找出来了… 唯一要注意的是输出原始题号而非排序后的题号，所以结构体中要存储原始题号）~

具体实现

朴素

把 kkk 从 0∼n0 \sim n0∼n 遍历一遍以确定 kkk 的大小。

复杂度 O(n2)\mathcal{O}(n^2)O(n2) ，看看数据范围就知道多半要超时，我就没写了（逃

二分搜索

由朴素很容易就能想到二分搜索。我们只需要对 kkk 在 0∼n0 \sim n0∼n 间进行二分搜索就好了，这样的复杂度是 O(nlogn)\mathcal{O}(nlogn)O(nlogn)。
