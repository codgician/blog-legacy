B: Bullet

二分图匹配 + 二分答案。

由每行、每列只能选一个怪物打，很容易想到将到二分图匹配的模型。我们把 NNN 行和 NNN 列分别看作 NNN 个行节点和 NNN 个列节点，并且分别把它们看作二分图的左部和右部。

首先，我们能很容易地算出最大匹配数。只要 Aij>0A_{ij} > 0Aij​>0（即怪物存在），我们就在左部 iii 节点和右部 jjj 节点间建一条边。接着，我们跑一遍匈牙利匹配就可以算出最大匹配数 maxAns\text{maxAns}maxAns。

接下来，我们来二分答案。记 AijA_{ij}Aij​ 中最小的非零经验值为 LLL，最大的经验值为 RRR，那么显然答案区间即 [L,R][L, R][L,R]。对于当前要判断可行性的最大经验值 midmidmid，我们只对 Aij≥midA_{ij} \ge midAij​≥mid 的点建边。由于题目要求的是在打怪数量最多前提下的最大经验值，我们只需要跑一遍匈牙利算法看一下在当前图中最大匹配数是否等于 maxAns\text{maxAns}maxAns 即可。

复杂度：O(N2log⁡N)\mathcal{O}(N^2\log{N})O(N2logN)

C: Cities

贪心。

既然要建连通图，那么显然每个点的点权至少要被算上一次（因为它们都一定需要与另一个点相连）。我们希望与每个点相连的点的权值都最小，那么我们只需要找出权值最小的点，并让剩下的点都与它相连就好了。

D: Dance

DFS + 贪心。

要想让总魔法值最多，我们可以考虑尽量多选能带来最多魔法值的基础魔法来进行升级。我们可以首先对树 DFS 一次，算出并记录每种基础魔法（也就是叶子节点）升级到最高魔法所能带来的魔法值。接下来我们对它从大到小进行排序，依次模拟升级过程（又是一次 DFS）。

这个做法貌似不能称作正解，因为如果构造一棵 ” N2\frac{N}{2}2N​ 个节点组成一条链，剩下 N2\frac{N}{2}2N​ 个节点分别是前 N2\frac{N}{2}2N​ 个节点的孩子“ 这样子的树，貌似可以把这个做法卡掉…… 还是出题人太良心了。🙈

E: Sequence

找规律。

要使删掉 aia_iai​ 后总好数最多，也就是要让删掉 aia_iai​ 对好数数量造成的影响最小。这里我们分两种情况讨论：

  • 如果删掉的 aia_iai​ 本身是好数，就意味着存在 k<i, ak<aik < i, \ a_k < a_ik<i, ak​<ai​。换句话说，对于 j>ij > ij>i 且 aj>aia_j > a_iaj​>ai​ 的好数，就算把 aia_iai​ 删掉了，依然有 k<j, ak<ajk < j, \ a_k < a_jk<j, ak​<aj​ 使得 aja_jaj​ 满足好数的定义。因此删掉 aia_iai​ 只会让总好数数量减一；
  • 如果删掉的 aia_iai​ 不是好数，那么删掉 aia_iai​ 会影响满足 j>i, aj>aij > i, \ a_j > a_ij>i, aj​>ai​ 且在 aja_jaj​ 之前只有 ai<aja_i < a_jai​<aj​ 的数。删掉 aia_iai​ 会让总好数数量减去满足那样条件的数的数量。

方法一

至此，一种显而易见的解决方式是使用树状数组。

我们可以开两个树状数组。第一个树状数组用于获取 aia_iai​ 之前有多少个数小于 aia_iai​，具体做法是读入 aia_iai​ 后就将 aia_iai​ 位置上的值改为 111，这样获取前 ai−1a_i - 1ai​−1 个位置的前缀和就是 aia_iai​ 之前小于 aia_iai​ 的数的个数了。我们记之为 pre[i]pre[i]pre[i]。

第二个树状数组用于获取 aia_iai​ 后满足上面第二类情况里提到的满足特定条件的数有多少个。具体做法是倒着遍历一次序列，如果对于 aia_iai​ 有 pre[i]=1pre[i] = 1pre[i]=1，就在树状数组中 aia_iai​ 的位置上加 111。如果 pre[i]=0pre[i] = 0pre[i]=0，就意味着 aia_iai​ 不是好数，我们获取第 ai+1a_i + 1ai​+1 个元素到第 nnn 个元素的后缀和就是满足特定条件元素的个数了。我们记之为 aff[i]aff[i]aff[i]。 当然如果 pre[i]=0pre[i] = 0pre[i]=0 的话就意味着 aia_iai​ 是好数，那么 aff[i]=1aff[i] = 1aff[i]=1（删去它对总好数数量的影响为 111）。

然后遍历一遍序列，找出 aff[i]aff[i]aff[i] 最小的 aia_iai​ 即可。

复杂度：O(Nlog⁡N)\mathcal{O}(N\log{N})O(NlogN)

方法二

如果进一步观察，我们可以意识到我们其实没有必要记录 pre[i]pre[i]pre[i]… 我们只需要记录 aia_iai​ 之前的最小值和次小值，同时扫一遍序列就行了。

我们记 minVal\text{minVal}minVal 为当前最小值，secMinVal\text{secMinVal}secMinVal 为当前次小值。

  • 如果 ai<minVala_i < \text{minVal}ai​<minVal，那么 minVal\text{minVal}minVal 变成了 aia_iai​，secMinVal\text{secMinVal}secMinVal 变成了 minVal\text{minVal}minVal；
  • 如果 minVal≤ai<secMinVal\text{minVal} \le a_i < \text{secMinVal}minVal≤ai​<secMinVal，那么意味着对于 aia_iai​ 在其之前且比 aia_iai​ 小的数只有 111 个。如果删去 minVal\text{minVal}minVal 必然会影响 aia_iai​，因此 aff[minVal]++aff[\text{minVal}]aff[minVal]++。同时，又由于 aia_iai​ 本身是个好数，因此删去 aia_iai​ 也会少一个好数，aff[ai]++aff[a_i]aff[ai​]++；
  • 如果 ai≥secMinVala_i \ge \text{secMinVal}ai​≥secMinVal，那么 aia_iai​ 本身是个好数，删去它会少一个好数，所以 aff[ai]++aff[a_i]++aff[ai​]++。

然后遍历一遍序列，找出 aff[i]aff[i]aff[i] 最小的 aia_iai​ 即可。

G: Game

变形的限制物品个数的统计方案数的背包问题。

首先，我们知道 Nim 游戏先手必胜的局面为每堆石子数的异或和非 000。那么显然，后手必胜的局面就是每堆石子数的异或和为 000。那么本题实际上就是要求我们拿走不超过 ddd 堆石子使得剩下每堆石子数的异或和为 000。

其次，我们需要知道异或运算的一个性质：a⊕a=0a \oplus a = 0a⊕a=0。另外， a⊕b⊕a=ba \oplus b \oplus a = ba⊕b⊕a=b。放在本题中，就意味着本题变为了：选出不超过 ddd 堆石子，使得选出石子异或和与所有石子异或和相等。

这样一来，我们不妨把所有石子异或和 xorSum\text{xorSum}xorSum 视作背包容量，每堆石子视作物品，并且选用物品个数不能超过 ddd。这就成了一个变形的背包问题，只不过总容量从加法和变成了异或和。

我们记录 dp[i][j][k]dp[i][j][k]dp[i][j][k] 表示对于前 iii 件物品，从中选取 jjj 件物品且已选取物品异或和为 kkk 时的总方案数。不难写出如下状态转移方程：

dp[i][j][k]={dp[i−1][j][k]j=0dp[i−1][j][k]+dp[i−1][j−1][k⊕ai]j>0dp[i][j][k] =
\begin{cases}
dp[i - 1][j][k] & j = 0 \
dp[i - 1][j][k] + dp[i - 1][j - 1][k \oplus a_i] & j > 0 \
\end{cases}dp[i][j][k]={dp[i−1][j][k]dp[i−1][j][k]+dp[i−1][j−1][k⊕ai​]​j=0j>0​

至于初始化，我们初始化 dp[0][0][0]=1dp[0][0][0] = 1dp[0][0][0]=1。
