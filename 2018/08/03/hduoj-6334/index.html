题面

Kazari 超喜欢刷题。在她眼中，一切题目都可依难度分为三个等级：简单、中等、困难。她最近来到了一个有 nnn 个节点树形迷宫，树上的每一条双向边都是一道题。

在接下来的 mmm 天中，第 iii 天她会选择 sis_isi​ 和 tit_iti​ 两点，并从 tit_iti​ 走到 sis_isi​。在走的过程中她必须解决一路上遇到的所有问题。

在每天的开始，Kazari 可以解决任意难度的题目，但是一旦她解决了一道难题后她就会失去信心，并且在这天接下来的时间里只能解决简单题了。

她的执着感动了刷题之神（大雾），于是神在这 mmm 天中每天会让某一条边上的题目难度变简单一级（困难变成中等，中等变成简单，简单不变），并且这一改变是持久的。

现在 Kazari 想知道对于第 iii 天：

  1. 她能否从 tit_iti​ 到达 sis_isi​？
  2. 在所有节点中，有多少个节点满足：从该节点出发可以到达 sis_isi​？

数据范围：

1≤n,m≤1051 \le n, m \le 10^51≤n,m≤105

∑n,∑m≤3×105\sum n, \sum m \le 3 \times 10^5∑n,∑m≤3×105

分析

首先感谢文末提及的参考博文！

树的特性

我们注意到地图是一棵树。

假设 ttt 和 sss 不存在祖先关系，我们考虑在树上从 ttt 到 sss 的过程：

  • 首先从 ttt 出发向上走到达 ttt 和 sss 的某个公共祖先 aaa；
  • 再从 aaa 向下走到达 sss。

而如果 ttt 和 sss 存在祖先关系：

  • 如果 sss 是 ttt 的祖先，向上走的过程中就到 sss 了，省略了第二步；
  • 如果 ttt 是 sss 的祖先，直接向下走就可以到 sss 了，省略了第一步。

由此我们看到，若 ttt 和 sss 之间可达，那么它们必然拥有共同的祖先。我们不难想到可以用并查集来维护这一属性。

并查集

那么我们不妨维护两个并查集：

  • f1(x)f_1(x)f1​(x) 表示从 xxx 出发向上只走难度为 111 的边最远可到达的点，也就是维护了难度为 111 的边的连通块；
  • f2(x)f_2(x)f2​(x) 表示从 xxx 出发向上只走难度为 111 或 222 的边最远可到达的点，也就是维护了难度为 111 和 222 的边的连通块。

另外，我们记录点 xxx 的父亲为 father(x)\text{father}(x)father(x)。

通过这两个并查集，我们甚至可以得知任一条边的难度值。假设已知 xxx 和 yyy 间一定存在边，那么如果 f1(x)=f1(y)f_1(x) = f_1(y)f1​(x)=f1​(y) 就说明这条边难度为 111；如果 f1(x)≠f1(y)f_1(x) \neq f_1(y)f1​(x)​=f1​(y) 但 f2(x)=f2(y)f_2(x) = f_2(y)f2​(x)=f2​(y) 就说明难度为 222，如果 f1(x)≠f1(y)f_1(x) \neq f_1(y)f1​(x)​=f1​(y) 且 f2(x)≠f2(y)f_2(x) \neq f_2(y)f2​(x)​=f2​(y) 则说明这条边难度为 333。

怎么初始化这两个并查集呢？首先既然题目给的是一棵无根树我们不妨任选一个点作为根节点然后 DFS，如果遇到某条连接 xxx 和 yyy 的边难度为 111，就在 f1f_1f1​ 和 f2f_2f2​ 里将它们按深度合并，如果难度为 222 就仅在 f2f_2f2​ 里将它们按深度合并。

问题 1

如果 ttt 可到达 sss，只存在如下三种情况：

  • f2(t)=f2(s)f_2(t) = f_2(s)f2​(t)=f2​(s)：这意味着只走难度为 111 或 222 的边就可从 ttt 到达 sss；
  • f1(father(f2(t)))=f1(s)f_1(\text{father}(f_2(t))) = f_1(s)f1​(father(f2​(t)))=f1​(s)：首先从 ttt 出发只走难度为 111 或 222 的边最远可到达 f2(t)f_2(t)f2​(t)。接下来我们走一条难度为 333 的边到达 father(f2(t))\text{father}(f_2(t))father(f2​(t))，然后只走难度为 111 的边到达 sss；
  • f2(father(f1(s)))=f2(t)f_2(\text{father}(f_1(s))) = f_2(t)f2​(father(f1​(s)))=f2​(t)：考虑到如果 ttt 是 sss 的祖先，应用上面一条规则就会出现问题：我们要走的难度为 333 的边不在 ttt 的上面而在 ttt 的下面。因此我们不妨将路径反过来考虑：从 sss 出发，开始只能走难度为 111 的边，经过一条难度为 333 的边后就可以走难度为 111 或 222 的边了。如果 ttt 和 sss 不存在祖先关系其实这一条和上一条是等价的。

问题 2

考虑从任意一点 xxx 可到达 sss 的条件（为了方便把路径反过来考虑）：从 sss 出发只能走难度为 111 的边，直到遇到并经过了一条难度为 333 的边后只能走难度为 111 或 222 的边，最终到达 xxx。

因此我们考虑把答案分为两部分考虑：

  • 从 sss 出发只经过难度为 111 或 222 的边可到达的点数；
  • 从 sss 出发先只经过难度为 111 的边，在经过 111 条难度为 333 的边，再经过难度为 111 或 222 的边可到达的点数。

我们对于 f1f_1f1​ 和 f2f_2f2​ 分别维护两个值 siz1siz_1siz1​ 和 siz2siz_2siz2​（注意这两个值都是对于并查集的祖宗而言的）：

  • siz1(f1(x))siz_1(f_1(x))siz1​(f1​(x)) 表示从 f1(x)f_1(x)f1​(x) 往下走，遇到第 111 条难度为 333 的边后只走难度为 111 或 222 的边能到达的点的个数；
  • siz2(f2(x))siz_2(f_2(x))siz2​(f2​(x)) 表示以 f2(x)f_2(x)f2​(x) 为祖宗的并查集的大小。

第一部分的答案，就是 siz2(f2(s))siz_2(f_2(s))siz2​(f2​(s))。

下面我们考虑第二部分。我们把第二部分分为两种情况：

  • 经过的难度为 333 的边在 sss 上方；
  • 经过的难度为 333 的边在 sss 下方。

对于第二部分的第一种情况，我们从 sss 出发向上走，判断第一条难度非 111 的边的难度是否为 333。如果难度为 333，我们记经过这条边后所处点为 kkk，那么答案就是 siz2(f2(k))siz_2(f_2(k))siz2​(f2​(k))。

对于第二部分的第二种情况，就是 siz1(f1(s))siz_1(f_1(s))siz1​(f1​(s))。

加起来就好了。
