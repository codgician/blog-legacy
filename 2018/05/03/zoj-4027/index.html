题面

现有一个由左括号 ‘(’ 和右括号 ‘)’ 两种共 nnn 个字符组成的字符串，每个括号都有一个权值 viv_ivi​。每次操作可以交换第 kkk 和第 k+1k + 1k+1 个括号，当且仅当第 kkk 个括号为左括号 ‘(’ 同时第 k+1k + 1k+1 个括号为右括号 ‘)’。刚开始时分数为 000，每次操作可获得分数 vk×vk+1v_k \times v_{k + 1}vk​×vk+1​。

试问经过任意次操作后可以得到的最大总分。

数据范围：
1≤n≤1031 \le n \le 10^31≤n≤103

−103≤vi≤103-10^3 \le v_i \le 10^3−103≤vi​≤103

分析

动态规划

首先我们不难发现交换具有以下特性：

  • 每一个左括号只可能往右走，因此不论如何变换，最初在第 iii 个左括号左边的右括号将永远在其左边；
  • 假设初始时第 iii 个左括号在第 jjj 个左括号的左边，那么不论如何变换第 iii 个左括号将永远在第 jjj 个左括号左边。这对右括号也成立。换句话说，左括号之间的相对位置不会发生改变，同时右括号之间的相对位置也不会发生改变。

也就是说，唯一变化的也就是左括号与右括号之间的相对位置。基于此，我们不妨用状态 dp[i][j]dp[i][j]dp[i][j] 来描述第 iii 个左括号在第 jjj 个右括号右边（且在第 j+1j + 1j+1 个右括号左边）时我们可得到的最大得分。既然第 iii 个左括号在第 jjj 个右括号右边意味着第 i+1i + 1i+1 以及以后的左括号都必然在第 jjj 个右括号右边，那么 dp[i][j]dp[i][j]dp[i][j] 显然可以由 dp[i][k] (j≤k≤rightNum)dp[i][k] \ (j \le k \le \text{rightNum})dp[i][k] (j≤k≤rightNum) 转移而来（其中 rightNum\text{rightNum}rightNum 代表右括号的个数）： 只需要再其基础上加上把第 iii 个左括号移动到第 jjj 个右括号右边所得的分数 scorescorescore 就行了 。于是我们不难得到状态转移方程：

dp[i][j]=max⁡{dp[i+1][k]+score}dp[i][j] = \max{ { dp[i + 1][k] + score } }dp[i][j]=max{dp[i+1][k]+score}

其中：

leftRightId[i]≤j≤rightNum, j≤k≤rightNum\text{leftRightId}[i] \le j \le \text{rightNum}, \ j \le k \le \text{rightNum}leftRightId[i]≤j≤rightNum, j≤k≤rightNum

需要说明的是， leftRightId[i]\text{leftRightId}[i]leftRightId[i] 代表第 iii 个左括号左边的与其最临近的右括号。显然，第 iii 个左括号不可能被交换到它最早所在位置的左边去（即第 leftRightId[i]\text{leftRightId}[i]leftRightId[i] 个右括号之前的右括号右边）。

接下来要解决的问题是 scorescorescore 如何计算。

我们注意到如果能将第 iii 个左括号移动到第 jjj 个右括号右边，那么第 iii 个左括号和第 jjj 个右括号之间必然全都是右括号。显然，如果我们要将第 iii 个左括号移动到第 jjj 个右括号右边，在此过程中必然需要先与其间的右括号发生移动。我们记第 iii 个左括号与第 jjj 个右括号之间（包含第 jjj 个右括号）的所有右括号权值之和为 sumsumsum，那么这个过程所获得的分数显然为（其中 leftVal[i]\text{leftVal}[i]leftVal[i] 指第 iii 个左括号的权值）：

score=leftVal[i]×sumscore = \text{leftVal}[i] \times sumscore=leftVal[i]×sum

至于 sumsumsum 如何计算，我们联想到第一条性质：第 iii 个左括号左边的右括号将永远在其左边。因此我们可以预处理右括号权值的前缀和 rightSum\text{rightSum}rightSum。由此，第 iii 个左括号和第 jjj 个右括号这个区间中的右括号也就等效于第 leftRightId[i]\text{leftRightId}[i]leftRightId[i] 个右括号（不包含）和第 jjj 个右括号之间的右括号。故 sumsumsum 可表示为：

sum=rightSum[j]−rightSum[ leftRightId[i] ]sum = \text{rightSum}[j] - \text{rightSum}[\ \text{leftRightId}[i] \ ]sum=rightSum[j]−rightSum[ leftRightId[i] ]

至此基本的转移方程式我们已经写出来了，接下来我们考虑边界。

首先，dp[i][j]dp[i][j]dp[i][j] 应当全部初始化为 −∞-\infty−∞，因为其中存在不合法的状态。于此同时，当 iii 为最右边的左括号时显然 dp[i+1][k]dp[i + 1][k]dp[i+1][k] 是不存在的，应取值为 000，所以在初始化的时候可以顺便将 dp[leftNum+1][k]dp[\text{\text{leftNum}} + 1][k]dp[leftNum+1][k] 全部初始化为 000。

如果第 iii 个左括号左边不存在右括号，我们可以记 leftRightId[i]\text{leftRightId}[i]leftRightId[i] 为 000。因此，dp[i][0]dp[i][0]dp[i][0] 实际上也是需要初始化的。

最后的答案也就是 dp[i][j]dp[i][j]dp[i][j] 中的最大值（严格地来说，应该是 dp[1][j]dp[1][j]dp[1][j] 中的最大值）。

该算法的复杂度级别为 O(N3)\mathcal{O}(N^3)O(N3)。

优化

我们注意到在上述实现中每次转移时都需要在 dp[i+1][k]dp[i + 1][k]dp[i+1][k] 中寻找最大值。于是我们思考，如果我们直接将状态 dp[i][j]dp[i][j]dp[i][j] 定义为第 iii 个左括号在第 jjj 个右括号右边（不论是否在第 j+1j + 1j+1 个右括号左边）时我们可得到的最大得分，那么这一层循环就可以省去了。由此我们可以写出状态转移方程：

dp[i][j]={max⁡{dp[i+1][j]+score,dp[i][j+1]}leftRightId[i]≤j≤rightNumdp[i+1][j]leftRightId[i−1]≤j≤leftRightId[i]−1dp[i][j] =
\begin{cases}
\max{ { dp[i + 1][j] + score, dp[i][j + 1] } } & \text{leftRightId}[i] \le j \le \text{rightNum} \
dp[i + 1][j] & \text{leftRightId}[i - 1] \le j \le \text{leftRightId}[i] - 1 \
\end{cases}dp[i][j]={max{dp[i+1][j]+score,dp[i][j+1]}dp[i+1][j]​leftRightId[i]≤j≤rightNumleftRightId[i−1]≤j≤leftRightId[i]−1​

其中，dp[i+1][j]+scoredp[i + 1][j] + scoredp[i+1][j]+score 代表将第 iii 个左括号恰好移到第 jjj 个右括号右边（且在第 j+1j + 1j+1 个右括号左边）后的最大得分，而 dp[i][j+1]dp[i][j + 1]dp[i][j+1] 则指第 iii 个左括号在第 j+1j + 1j+1 个右括号右边时的最大得分。二者中的最大值也就代表了第 iii 个左括号在第 jjj 个右括号右边时的最大得分。而当 leftRightId[i - 1]≤j≤leftRightId[i]−1\text{leftRightId[i - 1]} \le j \le \text{leftRightId}[i] - 1leftRightId[i - 1]≤j≤leftRightId[i]−1 时我们是无法将第 iii 个左括号恰好移动到第 jjj 个括号右边（且在第 j+1j + 1j+1 个右括号左边）的，因此此时 dp[i][j]=dp[i+1][j]dp[i][j] = dp[i + 1][j]dp[i][j]=dp[i+1][j]。至于 j<leftRightId[i - 1]j < \text{leftRightId[i - 1]}j<leftRightId[i - 1] 的情况，我们在推导 dp[i−1]dp[i - 1]dp[i−1] 的时候是用不到的，所以也不必计算。

至于 scorescorescore 的计算方法和预处理时的注意事项与上基本一致，就不作重复。唯一与上面不同的是，dp[i][j]dp[i][j]dp[i][j] 中这下子不存在不合法的状态了，所以可以方便地直接初始化 dp[i][j]dp[i][j]dp[i][j] 全部为 000。

优化后的复杂度级别为 O(N2)\mathcal{O}(N^2)O(N2)。

另一种转移方程

网络上的其它题解也提供了另一种与上文差别不大的转移方程的思路（参考 【ZOJ-4027】 Sequence Swapping(2018浙江省赛-D题））。

dp[i][j]dp[i][j]dp[i][j] 定义为第 iii 个左括号在第 jjj 个位置及其右边时我们可得到的最大得分，我们可得到如下转移方程式：

dp[i][j]={dp[i+1][j+1]+scorej=len−(leftNum−i)max⁡{dp[i+1][j+1]+score,dp[i+1][j]}leftPos[i]≤j≤len−(leftNum−i)−1dp[i][j+1]leftPos[i−1]≤j≤leftPos[i]−1dp[i][j] =
\begin{cases}
dp[i + 1][j + 1] + score & j = len - (\text{leftNum} - i) \
\max{ { dp[i + 1][j + 1] + score, dp[i + 1][j] } } & \text{leftPos}[i] \le j \le len - (\text{leftNum} - i) - 1 \
dp[i][j + 1] & \text{leftPos}[i - 1] \le j \le \text{leftPos}[i] - 1 \
\end{cases}dp[i][j]=⎩⎪⎪⎨⎪⎪⎧​dp[i+1][j+1]+scoremax{dp[i+1][j+1]+score,dp[i+1][j]}dp[i][j+1]​j=len−(leftNum−i)leftPos[i]≤j≤len−(leftNum−i)−1leftPos[i−1]≤j≤leftPos[i]−1​

其中，lenlenlen 代表字符串的长度，leftNum\text{leftNum}leftNum 代表左括号的个数。因此，leftNum−i\text{leftNum} - ileftNum−i 代表当前已经排号位置的左括号个数。对于已经排号的左括号我们自然不应再去更改它的位置，因此 j≤len−(leftNum−i)j \le len - (\text{leftNum} - i)j≤len−(leftNum−i)。另外 leftPos[i]\text{leftPos}[i]leftPos[i] 代表第 iii 个左括号的实际位置。

简而言之，第 iii 个左括号在第 jjj 个位置及其右边也就意味着第 i+1i + 1i+1 及以后的左括号都应在第 j+1j + 1j+1 个位置及其右边。由此，该转移方程也不难理解了。

这种算法的复杂度级别也为 O(N2)\mathcal{O}(N^2)O(N2)。
