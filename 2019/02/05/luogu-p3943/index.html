题面

现有一个长度为 nnn 的灯泡序列，其中 kkk 个位置上的灯泡是熄灭的，其余的是点亮的。我们可以进行 mmm 种操作，而第 iii 种操作是选定任意长度为 bib_ibi​ 的连续灯泡区间，对这个区间内所有灯泡的状态都进行一次翻转（亮变灭，灭变亮）。试求最少需要几次操作就可以让所有灯泡都被点亮。

数据范围：

1≤n,bi≤4×1041 \le n, b_i \le 4 \times 10^41≤n,bi​≤4×104

1≤m≤641 \le m \le 641≤m≤64

1≤k≤81 \le k \le 81≤k≤8

分析

差分思想

我们首先考虑如何快速对区间进行操作，而将区间操作转换为单点操作最常见的方法就是差分数组。不过这里我们面对的是一个 01 序列，我们可以把前缀和改成异或和，这样一来我们只需要翻转端点就可以达到区间修改的目的了。严格地说，记 aaa 代表原 01 序列，ddd 是原序列的异或差分数组。那么它们之间存在如下关系：

ai=b0⊕b1⊕⋯⊕bia_i = b_0 \oplus b_1 \oplus \dots \oplus b_iai​=b0​⊕b1​⊕⋯⊕bi​

如果我们需要翻转 aaa 中的 [l, r] 区间，只需要在 bbb 中翻转 lll 和 r+1r + 1r+1 两个位置即可。由此一来，我们实际上把 aaa 中的区间操作转换为了 ddd 中的单点操作（注意 ddd 的长度是 n+1n + 1n+1）。那么我们可以把原问题转换为这样一个新问题：给定一个 01 序列 ddd，一次只能选择两个位置上的值（这两个位置间隔的距离是题面给出的 mmm 种）并对其进行翻转，试问最少翻转多少次可以将序列翻转成目标序列。

我们不妨记灯亮为 000，灯暗为 111。这里这样记仅仅是为了方便，因为由此一来所有灯全亮时 aaa 为全 000，而其对应的 ddd 前 nnn 个位置也为全 000。至于 ddd 中的第 n+1n + 1n+1 个位置，不论其为 000 还是 111 都不会对 aaa 产生任何影响，因此第 n+1n + 1n+1 位可以是任意值。

需要注意的是，题目给出了 aaa 中 111 的个数 k≤8k \le 8k≤8，这一限制对应到差分数组 ddd 中会是怎样呢？如果我们要让 ddd 为 111111…111111\dots111111…，那么 aaa 的形式即如 101010…101010\dots101010…。不难发现若 aaa 中最多有 kkk 个 111，则 ddd 中最多有 2k2k2k 个 111。

完全背包

接下来我们来考虑转换后的问题。我们每次可以选择两个点进行翻转，为了让反转次数最少我们肯定不会选择两个 000 进行翻转，因为这样只会使得 111 的个数增多；如果我们选中两个 111，那么翻转后两个位置上的值都会变为 000。而如果我们选择一个 000 和一个 111 翻转，那么这次操作后 111 的个数并不会变化，其效果等效于将选中的两个位置上的值交换位置。进一步说，如果记两个位置间的距离为 Δx\Delta{x}Δx，则我们可以认为是 111 向左或是向右移动了 Δx\Delta{x}Δx。

那么问题可以进一步转换为：给定一个 01 序列，对于序列中每个 111，每一步可以向前或向后走 kkk 种距离，如果走到的位置上也是 111 则两者相消为 000，问最少需要多少步可以使得所有位置都变为 000。

若某一对 111 之间的距离为 Δx\Delta{x}Δx，则若要将这对 111 相消，就需要左右两端的 111 分别向左和向右移动到同一位置上。我们不妨把 Δx\Delta{x}Δx 看作背包容量，每种可能的步数看作背包里的物品（每种步数 vvv 要拆成 +v+v+v 和 −v-v−v 两个物品，因为既可以往左走又可以往右走），那么我们 O(nm)\mathcal{O}(nm)O(nm) 跑一个完全背包就可以得到消去每一对 111 所需要的最小步数。

一般图的最大匹配

在得到了消去每一对 111 所需要的最小步数后 www，我们可以考虑在这两个点间建立一条价值为 www 的无向边。这样一来，问题就转变为了一般图的最大匹配问题…… 大佬们可以使用带花树算法用 O(k3)\mathcal{O}(k^3)O(k3) 级别复杂度解决之…… 当然这道题数据范围很小，所以我们也可以使用状压DP来解决。

最后还有一个细节，前面说了 ddd 中第 n+1n + 1n+1 位可以是任意值，因此如果考虑这一位会带来一些不便。我们可以这样处理：如果位置 iii 经过若干步可以到达第 n+1n + 1n+1 位，我们就从 iii 向自己连一条边。其意义为 iii 可以与自己进行匹配。这样子就可以完全去掉第 n+1n + 1n+1 位从而简化模型。

为了减小状态数量，我们不妨只考虑所有 111 点。这样一来，初始状态用二进制表示就是 11…111\dots111…1，而目标状态即为 00…000\dots000…0。在 DP 过程中，我们不必要用 O(k2)\mathcal{O}(k^2)O(k2) 的复杂度去枚举每一个点对，因为那样会造成大量重复运算。事实上，我们可以每次选取最左的 111 点作为点对的一个点，然后 O(k)\mathcal{O}(k)O(k) 枚举另一个点直接进行转移即可。这样子其实是可以遍历到所有状态的（反正目标状态中这个 111 迟早要消去）。因此状压 DP 的复杂度为 O(k⋅2k)\mathcal{O}(k \cdot 2^k)O(k⋅2k)。

实现

总的复杂度为 O(nm+k⋅2k)\mathcal{O}(nm + k \cdot 2^k)O(nm+k⋅2k)。

这里是用完全背包+状压DP实现的，或许等我学会带花树算法后还会来诈尸。
